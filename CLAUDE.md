## Coding Rules & Workflow (MANDATORY — ปฏิบัติทุกครั้ง)

### Rules

| # | Rule | Trigger |
|---|------|---------|
| R1 | ถ้าไม่แน่ใจว่าเข้าใจคำสั่งถูก → ทวนคำสั่งก่อนลงมือ | ก่อนเริ่มทุกครั้ง |
| R2 | ก่อนแก้โค้ด → เช็คว่ากระทบส่วนอื่นที่ไม่ได้สั่งหรือไม่ แจ้งก่อนเสมอ | ก่อนแก้ทุกครั้ง |
| R3 | ถ้ามีวิธีที่ดีกว่า → แจ้งทันที | ระหว่างคิด |
| R4 | โค้ดสำคัญที่แก้ → ต้อง comment อธิบายไว้ | ระหว่างเขียน |
| R5 | เขียนโค้ดให้อ่านง่าย เป็นระเบียบ ไม่ติด Sonar issue | ระหว่างเขียน |

### Workflow (ทำตามลำดับทุกครั้ง)

1. **ก่อนเริ่ม** — ตรวจสอบความเข้าใจ ถ้าไม่แน่ใจ ทวนคำสั่ง `(R1)`
2. **ก่อนแก้** — เช็ค side effect แจ้งถ้ากระทบจุดอื่น `(R2)`
3. **แก้โค้ด** — ปฏิบัติตาม `R3` `R4` `R5`
4. **อัปเดต docs** — update `.md` ที่เกี่ยวข้องทันที **ไม่รอให้บอก**
5. **ตรวจ** — ตรวจ TypeScript error
6. **รายงาน** — ผ่าน Done Gate ทุกข้อแล้วถึงรายงาน

### Done Gate (ต้องผ่านทุกข้อก่อนรายงาน "เสร็จ")

- [ ] เข้าใจคำสั่งถูก และแก้โค้ดถูกจุด `(R1)`
- [ ] ไม่กระทบจุดอื่นที่ไม่ได้สั่ง หรือแจ้งไว้แล้ว `(R2)`
- [ ] update `.md` docs ที่เกี่ยวข้องแล้ว
- [ ] ไม่มี TypeScript error
- [ ] ไม่ติด Sonar issue `(R5)`
